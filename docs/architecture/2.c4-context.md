# C4 Context Diagram

## System Overview

**ft_transcendence** is a web-based page of sign up, login and profile that allows users to authenticate, edit profiles, through a secure web interface.

The system is designed as a containerized application with security-by-design principles and supports local deployment using a single command.

---

## Primary Users

### User
- Registers and authenticates
- Accesses the web interface
- Views profile

### Administrator
- Manages users and system configuration
- Monitors system health, performance metrics and security logs via Grafana

---

## External Systems

### Authentication Provider
- Supports third-party authentication (e.g. OAuth)
- Used as an external identity source

### Email Service
- Sends account-related notifications
- Used for verification and recovery flows

---

## Observability & Security Stack

### Vault
- Manages secrets, tokens and sensitive configuration.

### ModSecurity (WAF)
- Provides Web Application Firewall protection and request filtering.

### Prometheus
- Collects and stores real-time metrics from all services.

### Grafana
- Provides dashboards and alerts for system monitoring.

---

## System Boundary

The **ft_transcendence Platform** is the core system and includes:
- Web frontend
- Backend services
- Data storage
- Security, WAF and networking components

All internal components are isolated within a containerized environment.

---

## Trust Boundaries

- Users interact with the system exclusively through the web interface
- External systems communicate with the platform via secure, well-defined APIs
- Sensitive data (credentials, tokens) never leaves trusted system boundaries unencrypted

---

## Context Summary

The ft_transcendence system provides a secure, modular platform focused on core authentication and autorization functionality, serving as a foundation for future expansion.
